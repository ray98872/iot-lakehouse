01

Architecture

API parity, not approximation

MinIO speaks the real S3 API, so the producer uses unmodified boto3 and Spark uses the standard s3a:// connector with production Hadoop settings. Repointing the whole system at AWS is an endpoint-and-credentials change — zero application code touched.

A medallion lake in miniature

Raw, quarantine and processed zones mirror the bronze/silver/gold layout: every stage is independently auditable and rerunnable, and bad records land in a dead-letter prefix instead of vanishing silently.

02

The pipeline

Ingest

Explicit schema over the raw JSON zone — no costly inference, and PERMISSIVE mode shunts anything nonconforming into a corrupt-record column.

bronze

Quality gate

Nulls, unparseable timestamps and physically implausible values are filtered and written to quarantine — auditable, never silently dropped.

dead-letter zone

Rolling baseline

Per machine, a trailing 20-reading window — excluding the current row — yields a rolling mean and standard deviation for each metric.

silver

3σ scoring

Readings deviating >3σ from their own machine's baseline are flagged, z-scored, and written back as date-partitioned Parquet.

gold

Why 3σ, why per machine

Each machine is judged against its own recent history, so a naturally hot machine isn't a false alarm and a quietly degrading one can't hide in the fleet average. No labelled failure data, no training phase — it works from the first hour of telemetry, and becomes the benchmark any future ML model has to beat.

The business case

Unplanned industrial downtime costs tens of thousands of pounds per hour. Bearing wear and overheating show up as vibration and thermal drift long before failure — catching the 3σ excursion early converts an emergency stoppage into a scheduled maintenance slot.

Zero-cost iteration where it matters

The expensive part of cloud data engineering is the trial-and-error loop — every schema tweak, partition experiment and window rewrite billed per DBU and per GB. Here that loop runs at £0: every architectural decision is tested locally exactly as it would execute in production, and a dev cluster left idling overnight costs nothing instead of hundreds a month.

The skills transfer 1:1

PySpark window functions, S3A configuration, Hive-style partitioning, medallion zoning and Compose orchestration are the same muscles used on the real AWS + Databricks stack. The architecture is identical — only the bill is different.
